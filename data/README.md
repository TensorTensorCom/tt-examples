# Sample data

A deterministic 1k-vector binary file for testing the TensorTensor build API end-to-end without sourcing your own embeddings.

## Files

| File                    | Vectors | Dimensions | Size    |
|-------------------------|---------|------------|---------|
| `sample-1k-768d.bin`    | 1000    | 768        | ~3 MB   |

Generated by `generate-sample.js` with a fixed seed, so the output is reproducible byte-for-byte.

## File format

Raw `float32`, row-major, native byte order (little-endian on all common platforms).

```
[v[0][0], v[0][1], ..., v[0][767], v[1][0], v[1][1], ..., v[999][767]]
```

To recover the vector count from the file size:

```
rows = fileSize / (dimensions * 4)
```

## Using the sample with the TensorTensor API

The build API needs a publicly reachable HTTPS URL. Once this folder is mirrored to the public examples repo, the file can be referenced as:

```
https://raw.githubusercontent.com/TensorTensorCom/examples/main/data/sample-1k-768d.bin
```

Submit it like any other vectors file:

```bash
curl -X POST https://api.tensortensor.com/api/v1/build \
  -H "Authorization: Basic key-xxxxxxxxxxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "email": "you@example.com",
    "vectorsUrl": "https://raw.githubusercontent.com/TensorTensorCom/examples/main/data/sample-1k-768d.bin",
    "dimensions": 768
  }'
```

## Loading the file locally

The same binary loads in Node.js as a Float32Array:

```js
const fs = require('fs');
const buf = fs.readFileSync('sample-1k-768d.bin');
const vectors = new Float32Array(buf.buffer, buf.byteOffset, buf.length / 4);
// vectors.length === 1000 * 768 === 768000
```

Or in Python:

```py
import numpy as np
vectors = np.fromfile('sample-1k-768d.bin', dtype=np.float32).reshape(-1, 768)
# vectors.shape == (1000, 768)
```

## Regenerating

```bash
node generate-sample.js
```

Output is reproducible (seed = 42).
